# 배선 자동화(AX) 개발 회의 요약: 원가 최적화 시뮬레이션

## 🎯 회의 목적
배선 자동화(AX) 시스템 개발을 위한 핵심 기능 논의:
- **원가 최적화 시뮬레이션** 기반 배선 최적화 알고리즘 및 플랫폼 구축
- 시뮬레이션 결과를 시각화 및 검증하는 방안 모색

## 🧠 핵심 논의 및 분석

### 1. 프로그램 코어 기능 및 데이터 구조
| 구분 | 내용 |
|:--:|------|
| **Input** | 경량화된 '모델 정보' + OID(개체 ID) 기반의 '연결 정보' 2가지 데이터 기반 |
| **Core** | 데이터를 기반으로 배선 경로를 **계측(Measurement)** |
| **Output** | 시각화된 배선 경로(라우팅) + **원가 리포트** 도출 |

### 2. 최적화의 목적 분리 (중요!)
- **장비 이동을 통한 원가 최적화(시뮬레이션)** vs **케이블 라우팅 자체의 최적화**
- 두 기능이 완전히 다르므로 **인위적 합치기 금지** (어중간한 프로그램이 될 수 있음).
- **독립된 3개 타입**으로 분리하여 진행:
  1. 원가 중심 타입
  2. 생산비 중심 타입
  3. 합산형

### 3. 할증율·할인율 Faktor 모델 📈
$$ \text{배선 원가} = \text{재료비 (길이 \times 단가)} + \text{노무비 (Factor 적용)} $$

| Factor | 설명 |
|:--:|------|
| **할증률** | 케이블이 많이 꺾일수록 배선 난이도 증가 → 원가 상승 |
| **할인율** | 케이블을 묶어(Group) 포설할 때 효율 증가 → 원가 절감 |

### 4. 작업 지시서 및 인간 포설 기준 🛠️
- 현장 기준 반영: 곡선부, 장비 분조 등에 필요한 **인력 수 자동 계산**
- **목표:** 케이블 수량이 많아 수기로 계산 불가한 경우, 케이블별 작업 순서, 스케줄, 자동화 장비 투입 계획 자동 산출

## 🗓️ 향후 일정 (POC 계획)

| 기간 | 단계 | 비고 |
|:--:|:---|------|
| **7월 ~ 8월** | 요구사항 정립 | 라우팅, 생산성 요구사항 및 핵심 기능 정의 |
| **10월 ~ 11월** | POC 착수 | 연구 과제 연동 또는 서부 과제와 연동한 배분 알고리즘 POC |
| **내년 1월~2월** | 검증 및 고도화 | 3~4개월의 POC 결과 검증 후 본 개발(R&D) 진행 여부 결정 |

## 💡 효과성 및 기타 사항
- **CAD 팀 연동:** 현재 CAD 여력 부족 → API 등 우회 접근 (설계 데이터 추출 → 플랫폼 연동)
- **노력 절감:** 시스템 정상 착시 시 **연간 약 11만 Men-hour(MH)** 의 공수 절감 기대 (자산 최적화)
