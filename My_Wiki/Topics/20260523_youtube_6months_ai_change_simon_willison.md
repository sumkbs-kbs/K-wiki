## 핵심 요약

- **2025년 11월**은 최근 LLM 변화의 기준점이 됐고, 코딩 에이전트의 실용화와 노트북 실행 모델의 약진이 핵심
- Claude Sonnet 4.5 이후 **GPT-5.1**, Gemini 3, Claude Opus 4.5가 빠르게 경쟁했고, Opus 4.5가 몇 달간 선두로 보였음
- OpenAI와 Anthropic의 **검증 가능한 보상 기반 강화학습**은 Codex와 Claude Code 같은 하네스에서 코드 품질 향상으로 드러남

- Gemma 4, GLM-5.1, Qwen3.6-35B-A3B 같은 **오픈 가중치 모델**은 프런티어보다 약해도 기대를 크게 넘기 시작함

## 6개월을 가른 두 흐름

### 1. 코딩 에이전트의 실용화
- 11월에는 코딩 에이전트가 "종종 작동"하는 수준에서 **"대체로 작동"** 하는 수준으로 넘어감
- 사용자가 대부분의 시간을 어리석은 실수 수정에 쓰지 않아도 실제 작업을 맡길 수 있는 **일상 도구** 수준에 도달함
- OpenAI와 Anthropic은 2025년 대부분을 모델이 작성하는 코드 품질을 높이기 위한 검증 가능한 보상 기반 강화학습(RLVR)에 투입함
- 이 개선은 **Codex**와 **Claude Code** 같은 에이전트 하네스와 결합될 때 특히 두드러짐

### 2. 노트북 모델의 기대 초과
- 노트북에서 실행 가능한 모델들이 프런티어 모델보다 약하지만 기대를 크게 뛰어넘기 시작함
- 4월 기준 Qwen3.6-35B-A3B(20.9GB)가 노트북에서 Claude Opus 4.7보다 나은 펠리컨 그림을 그림
- "자전거를 탄 펠리컨"이 유용한 벤치마크로서의 한계를 이미 넘어섬

## 휴가철 실험과 과열 (2025.12-2026.1)

- 많은 사용자가 휴가 기간에 새 모델과 코딩 에이전트 실험
- **micro-javascript** — MicroQuickJS를 Python으로 포팅한 JS 구현체
- 브라우저 안에서 JavaScript → Python(Pyodide) → WebAssembly → JavaScript의 계층적 실행
- 흥미로운 결과물이지만: **버그가 있고 느리고 안전하지 않은 반쯤 완성된 JavaScript의 Python 구현체**
- 실제 필요성은 제한적 — 같은 기간 다른 프로젝트들도 조용히 은퇴

## 개인 AI 시스템 구축에 대한 시사점

### 1. 코딩 에이전트는 이제 "일상 도구"
- 6개월 전: "종종 작동"
- 현재: "대체로 작동"
- **→ 코딩 작업은 에이전트에 맡기고 QA만 하면 되는 시점**

### 2. 로컬 모델의 급성장과 프런티어 격차 축소
- 20.9GB 모델이 프런티어(Opus 4.7)를 능가하는 결과물 생성
- **→ 개인 머신(내 M5 Max, 128GB)에서 충분히 실용적인 모델 사용 가능**

### 3. 개인 AI 비서(Claws)의 부상
- OpenClaw가 "디지털 반려동물"로 자리잡기 시작함
- Mac Mini가 품절될 정도로 관심
- **→ Hermes + OpenClaw 조합으로 personal AI nonoc 구축은 올바른 방향**

### 4. 강화학습(RLVR)이 코드 품질의 핵심 동력
- OpenAI와 Anthropic이 RLVR에 집중
- **→ 모델 선택 시 RLVR 여부 중요해짐** (Qwen3.6 시리즈에 RLVR 적용 여부 확인 필요)
