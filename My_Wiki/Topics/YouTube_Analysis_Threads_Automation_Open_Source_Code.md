## 핵심 요약

노마드 코알라(투더제이 TTJ)가 **무료 공개한 스레드(Threads) 완전 자동화 프로그램** 분석 영상. 크몽 등에서 100만 원에 판매되는 상용 제품과 달리, **100% 오픈소스 + 커스터마이징 가능**이라는 것이 핵심 가치. Node.js 기반이며 Gemini/Codex/Claude CLI 기반 AI 모델 연동, 자동 좋아요·리포스트·댓글·포스팅, 스팸 필터, 언어 감지, 페르소나 기반 콘텐츠 생성 등 프리미엄 기능을 포함한다.

## 핵심 내용

### 주요 도구/기술

- **스레드 자동화 프로그램 (오픈소스, 무료)** — Node.js 기반 자동화 스크립트
  - `.env` 파일 기반 설정 (이메일·비밀번호·API 키)
  - `run.js` 메인 실행 파일, 146줄 이상에 AI 통합 로직 포함
  - **다운로드**: 투더코딩클래스 카페 → "스레드 자동화 코드 신청" 게시판 → 가입 인사 후 비밀번호 발급
  
- **지원 AI 모드 (CLI / API)**
  | 모드 | 번호 | 특징 | 비용 |
  |------|------|------|------|
  | Gemini CLI | 1 | 비대화형 1회성 호출 | 구독 토큰 사용 |
  | Codex CLI | 2 | 비대화형 1회성 호출 | 구독 토큰 사용 |
  | Claude Code CLI | 3 | 비대화형 1회성 호출 | 구독 토큰 사용 |
  | Gemini API | 4 | 실시간 API 호출 | 유료 |
  | OpenAI API | 5 | GPT API 호출 | 유료 |

  > **핵심 전략**: CLI 모드는 기존 구독 토큰을 재사용하므로 **추가 비용 제로**. API 모드는 별도 결제 필요.

- **세컨드 AI 모드**: 1차 AI 서버 불안정 / 토큰 소진 시 자동 폴백
- **스팸 필터**: AI가 게시 본문 분석 → 만남 요청 / 설문 / 폭력적·성인·사기성 → 자동 스킵
- **언어 감지**: 컨트리 랭 → 본문의 언어 감지 → 댓글 생성할 때 동일 언어 사용 (한국어/영어/일본어 외)
- **고정 댓글**: 코멘트 모드=1시 고정 멘트 사용 (후킹 문 + 실제 내용 댓글 분리 패턴)

### 핵심 변수 설정 (run.js 상단)

| 변수 | 값 | 의미 |
|------|---|------|
| 랜 모드 | 1: 수동 종료 / 2: 자동 종료 | 작업 후 브라우저 상태 |

| 코멘트 모드 | 0: 비활성 / 1: 고정 / 2: AI 생성 | 댓글 생성 방식 |

### 실행 과정

1. `npm install` 의존성 설치
2. `node run.js` 실행 → 브라우저 자동 열림
3. 수동 로그인 (인스타그램 연동)
4. 첫 번째 탭에 Threads URL 붙여넣기 → 로그인 감지
5. 자동 실행: 게시 스캔 → 스팸 필터 → 좋아요 → 리포스트 → 댓글 → 포스팅

### 주의사항 (중요)

- ⚠️ **메인 계정으로 자동화 금지** — 서브 계정 테스트 권장
- ⚠️ **새 계정 리스크 높음** — 2개 중 1개는 하루 만에 정지 가능
- ⚠️ 수동 좋아요조차 고강도로 여러 누르면 본인 인증 창 유도
- ✅ **핸드폰 인증 완료 계정**이 상대적으로 안전

### 수익화 방안

1. **스레드 계정 키운 후 제휴 마케팅 수익** — 자동화 → 팔로워 → 트래픽 → 수익

## 핵심 인사이트

1. **CLI 모드의 무료性は 절대적**. 기존 Gemini/Codex/Claude Code 구독 토큰을 그대로 재사용 — 추가 비용 0원. 이건 실제 비용적으로 큰 메리트다.
2. **비대화형(Asynchronous) 모드가 핵심**. 1회성 호출 방식으로 각 작업(좋아요·댓글·포스트)마다 AI를 필요할 때만 호출 → 토큰 효율 + 속도가 좋음.
3. **스레드 알고리즘이 현재 가장 유리한 시기**. 팔로워 수보다 콘텐츠 반응에 더 가중치 → 꾸준히 올리는 사람이 압도적 우위. 인플루언서 부족 = 신규 진입자 기회.
4. **페르소나 기반 AI가 필수**. AI 냄새 나는 자동 글은 반응 저조. 페르소나 + 내 말투 입력 → 자연스러운 콘텐츠가 핵심 성공 조건.
5. **100% 오픈소스 = 무한 확장성**. 코드 전체 공개 = 커스터마이징 자유, SaaS화 가능, 수익 파이프라인 다양화 가능.
6. **계정 안전이 가장 큰 리스크**. Threads의 자동화 감지 알고리즘이 매우 강력 — 메인 계정 사용 시 정지 가능성 높음. 테스트 → 서브 → 메인 순으로 점진적 사용 권장.
