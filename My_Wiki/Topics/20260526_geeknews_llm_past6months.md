### 🏁 2025년 11월: 변곡점
- **コーディング 에이전트 실용화 임계점 통과** — "종종 작동" → "대체로 작동" 수준으로 도약
- 성공 요인: **좋은 실행 하네스** + **RLVR**(검증 가능한 보상 기반 강화학습), 순수 모델 파라미터 증가 아님
- **최전선 모델 경쟁 가속:** `Claude Sonnet 4.5` → `GPT-5.1` → `Gemini 3` → `GPT-5.1 Codex Max` → `Claude Opus 4.5`
- **OpenClaw / "Claws" 트렌드:** 개인 AI 어시스턴트 붐, Mac Mini 부족 현상

### 🐦 벤치마커의 진화: "펠리컨" 테스트
- **기원:** SVG 자전거 타는 펠리컨 생성. 훈련 데이터에 없으며 실제 물리/로그직 추론 테스트
- **확장:** Gemini 3.1 Pro 우수, GLM-5.1은 "전동 킥보드 타는 버지니아 오푸스름" 그리기

## 커뮤니티 인사이트

### Coding 에이전트: 과장 vs 현실
- **편불균형:** 생산성 상승 맥락 의존. 상세 아키텍처 문서, 단계별 기획, 반복 리뷰/테스트 필요
- **프로덕션 코드 간극:** 복잡하고 불규격한 코드베이스에서는 여전히 어려움
- **워크플로우 전환:** "교체"가 아닌 **"강력한 외골격(exoskeleton)"**作为定位

## 실용적 교훈

1. **모델보다 하네스에 우선:** 실체적 도약은 실행 환경 + RLVR에서 왔음
2. **AI 워크플로우 구조화:**아키텍처/기획/반복 리뷰에는 AI, 복잡/고위험 부분은 수동
3. **로컬/오픈 모델 활용:** 20GB급 모델이 노트북에서 구동되며 잘 정의된 작업에선 최전선 대응
4. **기대치 관리:** "전환점"은 지속적인 개선이 아닌 **임계점 도약**인 경우가 많음
5. **철저한 테스트:** AI 코드 생성은 엄격한 검증 필요
6. **트렌드 모니터링:** 벤치마커 chasing보다 실제UTILITY에 집중

## 요약

지난 6개월 LLM의 핵심은 **순수 모델 스케일링**이 아닌 **실행 환경 + RLVR**에서 나왔습니다. 특히 로컬 모델이 20GB 수준으로 가볍게 내려오면서 노트북에서도 실무 활용이 가능해졌으며, 이는 프라이버시·비용 측면에서 큰 의미입니다.
