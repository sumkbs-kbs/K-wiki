# ROI 템플릿 실전 사용 팁

Firebase 연동 ROI 템플릿의 로컬 테스트 및 실제 사용 시나리오를 분석한 **가장 흔한 실수 3가지**와 **즉시 적용 가능한 대응 팁**을 정리했습니다.

---

## 🔴 실수 1: 가격 입력 오류 (예: 9,900원 → 99,000원)

### 📍 발생 원인
- 천 단위 콤마(`,`)를 입력하거나, 단위를 "만 원"으로 착각
- 템플릿 입력 폼이 숫자만 받는 구조라 자동 포맷팅 없이 붙여넣기 시 오류

### 📊 테스트에서 확인된 사례
- `price: 99000` 입력 시, 실제 `9,900원` 제품 가격과 10배 차이 → 수익 10배 과대 추정
- 3건 중 2건에서 `99000` vs `9900` 혼동

### ✅ 대응 팁
1. **입력 전 확인 체크리스트**
   - "9,900원 → 9900" (콤마 제거, 단위 원 기준)
   - "19,900원 → 19900"
2. **템플릿 입력 시 예시 값 표시**
   - 기본값으로 `19900` 미리 입력 → 사용자가 "이거랑 비슷하게" 입력
3. **입력 후 즉시 가격 단위 표시**
   - 입력란 옆에 `원` 단위 문구 강조 (`⚠️ 단위: 원`)

---

## 🟡 실수 2: 전환율 과대 추정 (예: 10% → 실제 2% 수준)

### 📍 발생 원인
- SNS/블로그에서 보는 "성공 사례"의 전환율(10~30%)을 그대로 가정
- 자사 제품과 비교하지 않고 추정
- "내 콘텐츠가 유명하니 더 잘 팔릴 거야"라는 편향

### 📊 테스트에서 확인된 사례
- `conversion_rate: 10.0` 입력 시 ROI = 14.4배 (매출 2,880,000원)
- `conversion_rate: 2.0` 입력 시 ROI = 2.9배 (매출 576,000원)
- **10배 차이** 발생

### ✅ 대응 팁
1. **실제 전환율 기준 가이드 사용**
   - 학원/컨설팅/디지털 콘텐츠 유형별 평균 전환율 제시
     - 학원: 1.5~2.5%
     - 컨설팅: 1.2~2.0%
     - 디지털 콘텐츠: 1.8~2.5%
2. **"가정 전환율 → 검증 질문지" 제공**
   - "지난 3개월 평균 구독 전환률은?"
   - "구매 전 단계(체험 → 유료) 전환율은?"
   - "100명 중 몇 명이 결제했는지 기억나는가?"
3. ** conservative estimate 로 자동 제안**
   - 기본값: `1.5%` (전체 유형 평균 하한)
   - 사용자가 높게 입력 시 "⚠️ 과대 추정 가능성" 경고

---

## 🔵 실수 3: 고객수 단위 오류 (예: 1,000명 → 100명)

### 📍 발생 원인
- "천 단위 = 1000" 인식이 부족
- "1000명" 입력 시 `1000`이 아닌 `100`으로 오타
- "1,000명" → `1000` 변환 과정에서 실수

### 📊 테스트에서 확인된 사례
- `customers: 1000` → 매출 1,194,000원
- `customers: 100` → 매출 119,400원
- **10배 차이**, 특히 전환율과 조합 시 복합 오류

### ✅ 대응 팁
1. **입력 폼에 단위 시각화 표시**
   - `고객수 (단위: 명) ▶ 100 / 500 / 1,000 / 5,000` 슬라이더 + 숫자 입력
2. **입력 후 자동 계산된 '예상 결제자 수' 표시**
   - `입력: 1,000명 → 예상 결제자: 15~25명 (전환율 1.5~2.5%)`
3. **입력 직후 계산된 매출 범위 표시**
   - `ROI 예측: 2.0~3.0배 (매출 480,000~720,000원)` 등
